# 네이버 스토어 순위·가격 모니터링

스마트스토어의 네이버쇼핑 검색 순위와 가격 적정성을 자동 분석하는 프로그램입니다.
브라우저 화면에서 설정부터 실행·리포트 열람까지 모두 할 수 있습니다.
입력한 API 키와 수집 데이터는 **본인 컴퓨터에만** 저장됩니다.

## 설치 & 실행 (macOS)

1. 이 페이지에서 **Code(초록 버튼) → Download ZIP** 을 눌러 내려받고 압축을 풉니다
2. 폴더 안의 **`실행.command` 더블클릭**
   - 처음 열 때 "확인되지 않은 개발자" 경고가 뜨면:
     파일을 **마우스 오른쪽 클릭 → 열기 → 열기** (최초 1회만)
   - 최초 실행 시 환경 설치로 1~2분 걸립니다
3. 브라우저 화면이 자동으로 열립니다 — 이후는 화면 안내대로:
   - **처음 설정**: 네이버 API 키 + 스마트스토어 주소 입력 (스토어명 자동 인식)
   - **키워드 등록**: 고객이 검색할 만한 검색어 추가
   - **지금 수집하기** 버튼 → 완료되면 대시보드·엑셀 리포트 생성

### 네이버 API 키 발급 (무료, 5분)

1. https://developers.naver.com/apps 접속 → 네이버 로그인
2. "애플리케이션 등록" → 이름 아무거나 (예: 스토어모니터)
3. 사용 API에서 **"검색"** 선택
4. 환경: "WEB 설정", URL: `http://localhost` 입력 후 등록
5. 발급된 **Client ID / Client Secret**을 프로그램 첫 화면에 입력

## 무엇을 알려주나

- **검색 순위**: 키워드별로 우리 상품이 네이버쇼핑 몇 위에 노출되는지 (1,000위까지)
- **가격 적정성**: 같은 단위(낱개/박스 자동 구분) 경쟁 상품 대비 우리 가격 위치,
  최저가/평균가 대비 %, 적정/고가/저가 판정
- **경쟁 판매처 목록**: 어느 스토어가 얼마에 팔고 있는지 전체 리스트 (엑셀)
- **추이 기록**: 실행할 때마다 `data/history.csv`에 누적 → 순위·가격 변동 추적

## 판정 기준 (config.yaml에서 조정 가능)

- `fair_band_pct: 10` — 시장 평균 ±10% 이내면 "적정"
- `warn_vs_min_pct: 30` — 최저가 대비 +30% 초과면 "고가 주의"
- `unit_band_ratio: 2.2` — 우리 가격의 1/2.2~2.2배 밖 상품은 판매 단위가
  다른 것으로 보고 비교에서 자동 제외 (낱개/박스 뒤섞임 방지)

## 참고

- 순위 기준: 네이버쇼핑 검색 정확도순(광고 제외). 실제 화면과는 광고 칸 수만큼
  차이가 날 수 있습니다.
- API 호출량: 무료 하루 25,000회 — 키워드 1개당 최대 10회이므로 수백 개도 매일 가능
- 키·스토어 변경: 화면 우측 상단 "키·스토어 변경" 버튼
- 문제가 생기면 `settings.yaml`을 지우고 다시 실행하면 초기 설정부터 다시 시작됩니다

## 폴더 구성

| 파일 | 역할 |
|---|---|
| `실행.command` | 실행 진입점 (더블클릭) |
| `app.py` | 웹 화면 (설정·키워드·실행·리포트) |
| `monitor.py` | 수집 파이프라인 |
| `collector.py` | 네이버 쇼핑 API 수집기 |
| `analyzer.py` | 순위·가격 분석 (단위 필터 포함) |
| `reporter_excel.py` / `reporter_dashboard.py` | 엑셀 / 대시보드 생성 |
| `config.yaml` | 키워드·판정 기준 (화면에서 편집됨) |
| `settings.yaml` | 본인 API 키·스토어 (자동 생성, 공유 금지) |
